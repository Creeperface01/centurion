Detailed Description

Represents a file "context" or handle.

Note
This class differs slightly from other library classes in that it is owning, but it does not throw if the internal pointer can't be created, etc. This is because file operations are error-prone, so we want to avoid throwing a bunch of exceptions, for performance reasons.
Since
5.3.0
