template<typename Hint , typename UserData = void>
auto cen::add_hint_callback ( SDL_HintCallback  fun,
UserData *  userData = nullptr 
) -> hint_callback<Hint, UserData>
noexcept

Adds a callback to observe changes of the value of the specified hint.

A callback handle object is returned, which can be used to easily disconnect the callback later. This function can be used with any function object that is stateless, such as traditional function pointers and lambdas. The simplest way to add a callback is with a lambda and no explicit user data.

Note
The callback will be immediately invoked with the current value of the hint.
In a future version of centurion (that supports C++20), the signature of the function object will be dependent on the UserData type. Unfortunately, this isn't really doable with C++17. Since it requires default-constructible stateless lambdas.
Template Parameters
Hintshould one of the many hint types defined in this header. However, all it requires is that the type provides a static method that returns a czstring.
UserDatathe type of the user data, defaults to void.
Parameters
funthe function object that will be invoked when the hint is updated. The signature should be void(void*, czstring, czstring, czstring).
userDatathe user data to associate with the callback.
Returns
a handle to the added callback.

◆ set_hint()

template<typename Hint , hint_priority priority = hint_priority::normal, typename Value , typename = std::enable_if_t<Hint::template valid_arg<Value>()>>
auto cen::set_hint ( const Value &  value) -> result

Sets the value of the specified hint.

This method will only accept values that are related to the specified hint, supplying the wrong kind of value causes a compile-time error. See the related hint class for more details about the associated value type. However, whilst this method is type-safe, it doesn't ensure that correct values are specified for the hints.
